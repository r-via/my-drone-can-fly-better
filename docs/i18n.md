# Internationalisation

Five locales: English, French, Spanish, German, Chinese. Every user facing
string in the app, including the findings themselves, comes from a dictionary.

## Shape

French is the reference, split into four files under
[`src/lib/i18n/fr/`](../src/lib/i18n/fr/):

```
fr/
  index.ts    export const fr = { rules, lint, system, ui }
  rules.ts    one entry per flight rule, ~25 keys
  lint.ts     one entry per config rule, 9 keys
  system.ts   parser errors, worker progress, read errors
  ui.ts       layout, page, components, chart labels
```

The `Dict` type is derived from it:

```ts
export type Dict = {
  rules: typeof rules;
  lint: typeof lint;
  system: typeof system;
  ui: typeof ui;
};
```

Every other locale is a single file declaring `const xx: Dict = {...}`, so a
missing key, an extra key or a renamed key is a **compile error**, not a runtime
surprise. `npx tsc --noEmit` is the completeness check.

Note that the reference files deliberately avoid `as const`. Values must stay
typed as `string`, otherwise a literal type from the French file would forbid
any other text in the translations.

## Entries are values or functions

A key is either a plain string or a function taking the numbers it must
interpolate:

```ts
// fr/system.ts
sessionTooShort: (frames: string) =>
  `Session trop courte (${frames} frames) - probable blip d'armement`,

// fr/ui.ts
analyzeButton: (count: number): string =>
  count > 1 ? `Analyser les ${count} logs` : 'Analyser le log',
```

That keeps pluralisation and word order inside the translation, where a
translator can fix them, instead of in the calling code.

## Registry and detection

[`src/lib/i18n/index.ts`](../src/lib/i18n/index.ts):

```ts
type Locale = 'en' | 'fr' | 'es' | 'de' | 'zh';
LOCALES: ReadonlyArray<{ code: Locale; label: string }>   // drives the switcher
getDict(locale): Dict          // falls back to English on an unknown code
isLocale(v): v is Locale
detectLocale(): Locale               // browser
detectLocaleFromEnv(env): Locale     // Node
persistLocale(locale): void
```

**Browser detection**: `localStorage['mdcfb.locale']`, then the first two
characters of `navigator.language`, then English.

**Node detection** (used by the CLI): `LC_ALL`, `LC_MESSAGES`, `LANG`,
`LANGUAGE`, in that order, parsed POSIX style. `fr_FR.UTF-8`, `fr`, and a
gettext list like `fr:en_US` all resolve to `fr`. A `C` or `POSIX` locale falls
back to English.

## Client wiring

[`src/lib/i18n/locale.tsx`](../src/lib/i18n/locale.tsx) provides a React context:

```tsx
<LocaleProvider>  // wraps <Shell> in layout.tsx
const { locale, dict, setLocale } = useLocale();
```

The static export renders in English, then the real locale is applied after
hydration by an effect. The same effect keeps `document.documentElement.lang` in
sync, mapping `zh` to `zh-CN`.

`setLocale` writes to localStorage and updates state. `LanguageSwitcher` is a
plain `<select>` fed by `LOCALES`.

## Why changing language re-runs the analysis

Findings are generated as final strings **inside the worker**, in the requested
locale. There is no template left to re-render on the main thread. So when the
locale changes while a report is on screen, `page.tsx` re-runs the whole
analysis on the still selected `File` objects:

```tsx
if (lastReportLocale.current !== locale) {
  lastReportLocale.current = locale;
  if (files.length > 0) void startAnalysis();
  else analyzer.reset();
}
```

This costs a couple of seconds and keeps a single source of truth for a
finding's text. The alternative, shipping structured findings and formatting in
the UI, would move the interpolation logic out of the dictionaries and into
components.

If the files are gone, for instance after a page reload, the report is reset
rather than shown in a stale language.

## Charts are the exception

The three SVG chart components take their labels through a plain `labels` prop
rather than calling `useLocale()`. They are pure functions of their props, with
no hooks, which is what lets `tests/charts.test.ts` render and assert on them in
a Node environment.

`ReportView` passes `dict.ui.charts.spectrum`, `.step` and `.timeline` down.

## Adding a language

1. Copy `src/lib/i18n/en.ts` to `src/lib/i18n/<code>.ts` and translate it.
2. Register it in `src/lib/i18n/index.ts`: the `Locale` union, the `LOCALES`
   list with a native language label, the `REGISTRY` map, and the two detection
   helpers `isLocale` and `detectLocale`.
3. Run `npx tsc --noEmit`. It fails until every key exists.

Translate meaning, not words. These strings get read by a pilot standing in a
field with a dead pack, so keep them short and concrete.

Two conventions that are easy to miss:

- `ui.units.mega` / `ui.units.kilo` exist so file sizes read `Mo` / `Ko` in
  French and `MB` / `KB` elsewhere.
- The decimal separator is handled in `ReportView.makeFormatters`, not in the
  dictionaries: French gets a comma, every other locale a dot. Category slugs
  (`vibrations`, `filtres`, `moteurs`) stay French internally and are mapped
  through `dict.ui.categories`.
- No em dashes in any user facing string. Use a plain hyphen, a colon or a full
  stop.
